# デザインガイドライン

本アプリケーション（キギョヒカ）のデザイン原則と標準仕様をまとめました。
アプリケーション全体で統一感のあるUI/UXを提供するために、以下の定義に従ってください。

## 1. カラーパレット (Color Palette)

### ブランドカラー

アプリケーションの印象を決定づける主要なカラーです。

| 色名 | カラーコード | 用途 |
| :--- | :--- | :--- |
| **Deep Blue** (Primary) | `#0A3BA1` | メインカラー。ブランドロゴ、見出し、主要なアクションボタン、アクティブ状態など。信頼性と知性を表現します。 |
| **Electric Lime** (Accent) | `#C0F03C` | アクセントカラー。ハイライト、アイコンの一部、強調したい要素に使用し、デザインに先進性と活気を与えます。 |

### ニュートラルカラー & 背景

| 色名 | 用途 |
| :--- | :--- |
| **White** (`#FFFFFF`) | 背景色、カード背景。 |
| **Light Gray** (`bg-gray-50`) | アプリケーション全体の背景ベース。 |
| **Text Main** | 本文、主要なテキスト。視認性を確保するため黒に近いグレーを使用。 |
| **Text Muted** | 補足情報、ラベルなど。情報の優先度を下げる場合に使用。 |

### セマンティックカラー

- **Success**: 緑系 (ポジティブな数値など)
- **Destructive**: 赤系 (削除アクション、ネガティブな数値など)

---

## 2. タイポグラフィ (Typography)

フォントには可読性が高くモダンな **Geist** ファミリーを使用します。

### フォントファミリー

- **Sans-serif**: `var(--font-geist-sans)` - 通常のテキスト、見出し
- **Monospace**: `var(--font-geist-mono)` - コード、数値データなど

### テキストサイズと役割

- **見出し (Heading)**: 太字 (`font-bold`) を使用し、階層を明確にする。
- **本文 (Body)**: 視認性を重視し、適切な行間 (`leading-relaxed`) を確保する。
- **キャプション/補足**: `text-xs` や `text-sm` と `text-muted-foreground` を組み合わせる。

---

## 3. UIコンポーネント & スタイル (UI Components)

### 角丸 (Border Radius)

- **基本**: `rounded-xl` (または `var(--radius)` = `0.625rem`) を標準とし、少し大きめの角丸で親しみやすさを出します。
- **完全な円**: アイコン背景やカプセル型のUI（検索バー、タグ）には `rounded-full` を使用します。

### カード (Cards)

- **デフォルト**: 白背景、ボーダーなしまたは薄いボーダー、シャドウなし。
- **ホバー時**: `hover:shadow-lg` で浮き上がるインタラクションを追加します。
- **アクティブ時**: このアプリの特徴として、選択状態を `ring-2 ring-primary` で明確に表現します。

### ボタン (Buttons)

- **Primary**: `bg-primary text-white` - 最も重要なアクション。
- **Outline**: `border border-primary text-primary` - 二次的なアクション。
- **形状**: 基本的に `rounded-full` を好んで使用し、モダンな印象を与えます。

### エフェクト (Effects)

- **グラスモーフィズム (Glassmorphism)**: フローティングUI（検索バーなど）には、背景ぼかし (`backdrop-blur-xl`) と半透明背景 (`bg-white/90`) を使用します。
- **グラデーション**: ヒーローセクションなどの背景には、`from-primary/5` などの極めて薄いグラデーションを使用し、奥行きを作ります。

---

## 4. レイアウト & スペーシング (Layout)

- **コンテナ**: `container mx-auto px-4` を基本とし、PC画面では最大幅を制限して可読性を保ちます。
- **余白**: 要素間は `gap-4` (16px) や `space-y-6` (24px) などを基準とし、窮屈にならないよう十分なホワイトスペースを確保します。
